# OSTEP Ch4-5 进程 + 进程 API

> 2026-05-16

## 进程是什么

> 进程 ≠ 程序。**进程 = 程序代码 + 运行时状态**

运行时状态包括：地址空间 + 寄存器 + 打开的文件。
Linux 内核里用 `task_struct` 保存这些（在 `include/linux/sched.h`，几千行）。

## 地址空间：4 大件

| 段 | 存什么 | 增长方向 |
|----|--------|---------|
| Code（代码段） | 程序指令 | 固定 |
| Data | 全局 / 静态变量 | 固定 |
| Heap | `malloc` 分配的 | 向上长 ↑ |
| Stack | 函数调用 / 局部变量 | 向下长 ↓ |

堆和栈对着长，中间是空的——这样谁先撑爆都不影响对方。

## 进程的 4 种状态

- **Running**：正在 CPU 上跑
- **Ready**：准备好了，等 CPU
- **Blocked**：等 I/O 等事件
- **Zombie**：死了但还没被父进程 `wait` 收尸

        ┌─────────┐    被调度上 CPU      ┌─────────┐
        │  Ready  │ ────────────────→  │ Running │
        │ (就绪)   │ ←────────────────  │ (运行)   │
        └─────────┘    时间片到/被抢占    └─────────┘
             ▲                              │
             │                              │ 发起 I/O 等待事件
             │ I/O 完成、事件来了            ▼
             │                          ┌─────────┐
             └──────────────────────── │ Blocked │
                                       │ (阻塞)   │
                                       └─────────┘
                                            
        (进程退出后短暂)── Zombie ──> 被 wait 后彻底消失

> 跟 FreeRTOS task 的 Running/Ready/Blocked **一模一样**，多了个 Zombie

## 程序怎么变成进程（OS 干了啥）

1. **load**：把磁盘上的 ELF 加载到内存
2. **setup stack/heap**：初始化栈、堆
3. **setup fd**：建立默认文件描述符（0=stdin、1=stdout、2=stderr）
4. **jump to main**：跳转到 `main()` 开始跑

## 三大 API

### fork — 复制

- **调用一次返回两次**：父进程拿到子的 PID，子进程拿到 0
- 父子谁先跑**不确定**

### exec — 变身

- **成功后不返回**（当前代码被新程序完全替换）
- 常用 `execvp`（自动搜 `$PATH`，参数用数组）

### wait — 等待

- 不 `wait` → 子进程变 **zombie**
- 父进程先死 → 子进程变 **orphan**，被 PID=1 的 `init` 收养

## 为什么 fork 和 exec 分开？

**fork 之后、exec 之前**有个"窗口期"可以改东西：

- 改文件描述符 → 实现 `>` `<` 重定向
- 改环境变量、用户权限、信号处理

shell 实现 `ls > out.txt`：

1. `fork`
2. 子进程关掉 fd=1（stdout）
3. 子进程打开 `out.txt`（自动占 fd=1，因为 OS 总用最小可用 fd）
4. `exec ls`，ls 写 stdout 就写进了 `out.txt`

**Unix 这套设计 50 年没人能取代，因为它简单到极致**。

## STM32 对照
| | FreeRTOS task | Linux process |
|---|---|---|
| 状态机 | Running/Ready/Blocked | 同上 + Zombie |
| 创建 | `xTaskCreate(func, ...)` | `fork()` + `exec()` |
| 地址空间 | **共享**（同一片 RAM） | **独立**（虚拟内存隔离） |
| 通信 | 全局变量直接读写 | 必须用 IPC（管道/共享内存/socket） |
**最大区别**：地址空间是否独立。这是"进程"和"线程"的本质分界线（线程共享地址空间，进程不共享）。
